# Contributing to `waka`

Thank you for your interest in contributing! `waka` is a community-driven open source project and we welcome all contributions.

## Quick Start for Contributors

```bash
# 1. Fork the repo and clone
git clone https://github.com/YOUR_USERNAME/waka.git
cd waka

# 2. Install Rust (if needed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 3. Install development tools
cargo install cargo-nextest cargo-deny git-cliff
rustup component add rustfmt clippy

# 4. Verify everything builds and tests pass
cargo build
cargo nextest run

# 5. Create a branch
git checkout -b feat/my-feature
```

## Development Workflow

Read `CLAUDE.md` — it contains the authoritative development guidelines.

The short version:

1. Find a task in `DEVELOPMENT_PLAN.md` or pick up an issue
2. Read the relevant spec in `SPEC.md`
3. Implement with tests
4. Run the pre-commit checklist
5. Commit with a conventional commit message
6. Open a PR

## Pre-commit Checklist

```bash
cargo fmt --all                                    # Format
cargo clippy --all-targets -- -D warnings          # Lint (zero warnings)
cargo nextest run                                  # Tests (all green)
cargo doc --no-deps                                # Docs (no warnings)
```

## Commit Messages

We use [Conventional Commits](https://www.conventionalcommits.org/). See `CONVENTIONS.md §2` for the full reference.

```
feat(api): add leaderboard endpoint
fix(cache): handle missing cache directory gracefully
docs(readme): add tmux integration example
```

## Running Tests

```bash
cargo nextest run                    # All tests
cargo nextest run -p waka-api        # Tests for one crate
cargo test --doc                     # Documentation tests
cargo insta test                     # Snapshot tests
cargo insta review                   # Review changed snapshots
```

## Adding a New Command

1. Add the command struct to the appropriate subcommand in `crates/waka/src/cli.rs`
2. Add the handler function in `crates/waka/src/commands/`
3. Add the renderer in `crates/waka-render/src/`
4. Add snapshot tests in `crates/waka-render/src/`
5. Wire the handler in `crates/waka/src/main.rs`
6. Update `DEVELOPMENT_PLAN.md`

## Updating Shell Completions

Completions are generated at runtime by the `waka completions <shell>` command.
After changing the CLI structure, regenerate the static completion files:

```bash
cargo build
./target/debug/waka completions bash        > completions/waka.bash
./target/debug/waka completions zsh         > completions/_waka
./target/debug/waka completions fish        > completions/waka.fish
./target/debug/waka completions powershell  > completions/_waka.ps1
./target/debug/waka completions elvish      > completions/waka.elv
```

Commit the generated files alongside any CLI changes.

### Installing completions for development

```bash
# Bash
source <(waka completions bash)

# Zsh
waka completions zsh > ~/.zsh/completions/_waka
# (ensure ~/.zsh/completions is on your $fpath)

# Fish
waka completions fish > ~/.config/fish/completions/waka.fish

# PowerShell
waka completions powershell | Out-String | Invoke-Expression
```

## Updating Man Pages

Man pages are automatically generated by `build.rs` using `clap_mangen` whenever
`src/cli.rs` changes. After modifying the CLI structure:

```bash
# Regenerate man pages (runs automatically as part of cargo build)
cargo build -p waka
```

Generated files are written to `man/` (workspace root) and committed to the
repository. Each command and sub-command gets its own `.1` roff file:

```
man/waka.1               # top-level command
man/waka-stats.1         # stats sub-command
man/waka-stats-today.1   # stats-today leaf
# ... and so on for 45+ pages
```

To install man pages locally:

```bash
# Linux (user-local)
mkdir -p ~/.local/share/man/man1
cp man/*.1 ~/.local/share/man/man1/
mandb -q ~/.local/share/man

# macOS
mkdir -p /usr/local/share/man/man1
cp man/*.1 /usr/local/share/man/man1/
```

## Architecture
